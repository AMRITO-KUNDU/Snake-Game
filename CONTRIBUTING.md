# Contributing

Thanks for your interest in improving this project.

## Quick Start

1. Fork the repo
2. Create a feature branch
3. Make changes with small, focused commits
4. Test in a browser (see README for running a local server)
5. Open a pull request describing what changed and why

## Code Style

- Keep changes minimal and readable
- Prefer small helper functions over deeply nested logic
- Avoid adding build tooling unless it’s necessary for the feature

## Reporting Bugs

Include:

- Browser + version
- Steps to reproduce
- Expected vs actual behavior
- Screenshot / screen recording if relevant
